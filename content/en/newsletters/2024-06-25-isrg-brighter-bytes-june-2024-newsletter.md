									<h3><span style="color:#003a70;">Memory Safe NTP for Let’s Encrypt: Deploying ntpd-rs</span></h3>

									<p>We are thrilled to announce we’ve deployed ntpd-rs, the first piece of memory safe software from Prossimo that has made it into the Let’s Encrypt infrastructure.<br>
									<br>
									<strong><a href="www.memorysafety.org/initiative/ntp/" style="color:#3771c8;">Ntpd-rs</a> </strong>is a secure, memory safe implementation of the Network Time Protocol (NTP), which is critical for maintaining accurate system time across operating systems. The implementation has matured and is now maintained by Tweede golf’s <strong><a href="tweedegolf.nl/en/pendulum" style="color:#3771c8;">Project Pendulum</a></strong>.<br>
									<br>
									This deployment marks a crucial step in safeguarding our systems against vulnerabilities inherent in languages like C and C++. Learn more about the implementation and our plans for the future in our <strong><a href="letsencrypt.org/2024/06/24/ntpd-rs-deployment" style="color:#3771c8;">latest blog post</a></strong>.</p>

									<h3 style="line-height: 1.38; margin-bottom: 8px;"><span style="color:#003a70;">Encrypted Client Hello (ECH) Support for Rustls</span></h3>

									<p>We’re thrilled to share that the <strong><a href="github.com/rustls/rustls" style="color:#3771c8;">Rustls TLS library</a></strong>&nbsp;now has experimental support for client-side Encrypted Client Hello (ECH).<br>
									<br>
									When client software connects to a server, it uses an IP address typically obtained via DNS. Servers hosting content for multiple domain names use the Server Name Indication (SNI) extension in the TLS connection request to specify the desired domain name, which is not encrypted by default.<br>
									<br>
									ECH is a proposed Internet standard to encrypt the domain name, enhancing privacy by preventing network observers from identifying the intended website. By enabling ECH support, we’re continuing to position Rustls as a viable memory safe alternative to OpenSSL.&nbsp;​​​​​<a href="www.memorysafety.org/initiative/rustls/" style="color:#3771c8;"><strong>Read the blog post</strong></a>.</p>

									<h3 style="line-height: 1.38; margin-bottom: 8px;"><span style="color:#003a70;">Increasing domain validation security from BGP attacks</span></h3>

									<p>Thanks to a recent grant, we’re excited to continue our partnership with Princeton University’s Center for Information Technology Policy (CITP) team.<br>
									<br>
									To date our work with Princeton has focused on defending against BGP attacks on domain control validation via Multi-Perspective Issuance Corroboration (MPIC). Instead of validating from one network perspective, we validate from multiple perspectives as well as from our own data centers.&nbsp;<br>
									<br>
									This year Let's Encrypt is adding two new remote perspectives for domain validation, totaling five validation requests. This enhancement provides more robust domain validation security, improving visibility and protection against BGP attacks.<br>

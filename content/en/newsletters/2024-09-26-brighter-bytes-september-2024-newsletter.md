									<h3><span style="color:#003a70;">Let's Encrypt Growth</span></h3>
									The number of websites protected by Let’s Encrypt certificates has seen consistent growth over the past nine years. In 2022, the number of secured websites grew by 16.7%. That momentum continued in 2023 with an 18.6% increase. Now, just three-quarters of the way through 2024, we’ve already seen a remarkable 26.7% growth—more than 100 million new websites supported by Let's Encrypt.<br>

By enabling more websites to adopt strong, up-to-date encryption, we’re helping to ensure a safer and more secure online experience for all users.<br>

									<h3><font color="#003a70">Eliminating Memory Safety Vulnerabilities at the Source</font>​​​​​</h3>
									Yesterday Google published a new Security Blog post highlighting the positive, measurable benefits of transitioning to memory safe languages, including an incredible reduction of memory safety vulnerabilities in Android from 76% to 24% in just six years. By focusing on making new code memory safe, they've made huge strides towards our shared goal of eliminating this entire class of security vulnerabilities.&nbsp;<a href="https://outreach.abetterinternet.org/e/1011011/y-vulnerabilities-Android-html/2rgy7/%%email_id%%/h/%%internal.tracker_url_hash%%" style="color:#3771c8;">Read more</a>

									<h3><font color="#003a70">Big Steps Forward for River</font></h3>
									Just about every significant deployment on the Internet makes use of reverse proxy software, and the most commonly deployed software is not memory safe.<br>
									<br>
									Written in Rust for memory safety, River aims to outperform existing C/C++ options like Nginx. Since its launch earlier this year the River reverse proxy has made great progress. Beyond basic functionality, River now includes:
									<ul>
										<li>Load Balancing to spread traffic efficiently</li>
										<li>Rate Limiting for anti-abuse and load control</li>
										<li>KDL-based Configuration for cleaner setup</li>
										<li>Static File Support for direct file serving</li>
										<li>Graceful Reloads without interrupting connections</li>
										<li>CIDR Blocking to stop malicious traffic fast</li>
									</ul>
									While not production-ready yet, we’re inviting early feedback and gearing up for more exciting features like ACME support and WASM extensibility.&nbsp;<a href="https://outreach.abetterinternet.org/e/1011011/blog-river-release-/2rgyb/%%email_id%%/h/%%internal.tracker_url_hash%%" style="color:#3771c8;">Read more</a>

									<h3><font color="#003a70">Bringing Memory Safety to AV1 Video Decoding</font>​​​​​</h3>
									AV1 is an increasingly important video format and it needs a memory safe, high performance decoder. We’ve been working with the team at Immunant to develop rav1d, a Rust-based port of dav1d, a C decoder.<br>
									<br>
									In a two-part blog series, Immunant CTO and Co-Founder Stephen Crane details the initial migration and safety improvements as well as the optimization work for speed and performance.<br>

<h3><span style="color:#003a70;">Introducing Sunlight!</span><img align="right" alt="" height="200" src="/images/newsletters/2024/2024-03-21-isrg-brighter-bytes-march-2024-newsletter/image-03.png" style="width: 180px; height: 200px; margin: 0px; float: right;" width="180"></h3>

<p>Sunlight is a <strong>new Certificate Transparency (CT) implementation</strong>, designed with the evolving demands of the modern Web PKI in mind. It was built in collaboration with <strong>Filippo Valsorda</strong> and with feedback from the broader community, including Google's Chrome and TrustFabric teams, the Sigsum project, and other CT log and monitor operators.&nbsp;<br>
<br>
Sunlight aims to address the scalability and reliability challenges faced by current CT logs due to the ever-increasing volume of certificates issued daily (4 million by Let’s Encrypt alone!). Sunlight's architecture departs from traditional relational database storage, opting instead for a more scalable, efficient, and cost-effective static tiles system for storing certificates. This approach significantly improves the log's performance and reliability.<br>
<br>
<strong><a href="letsencrypt.org/2024/03/14/introducing-sunlight" style="color: rgb(55, 113, 200); text-decoration: none;">Read the full blog post</a></strong> to learn how Sunlight is illuminating the path for a better Internet.</p>

									<h3 style="line-height: 1.38; margin-bottom: 8px;"><span style="color:#003a70;">River Reverse Proxy&nbsp;</span></h3>

									<p>Earlier this month, we <strong><a href="www.memorysafety.org/blog/introducing-river/" style="text-decoration: none; color: rgb(55, 113, 200);"><span style="color:#3771c8;">announced&nbsp;the launch of </span></a><a href="www.memorysafety.org/blog/introducing-river/" style="text-decoration: none; color: rgb(55, 113, 200);"><span style="color:#3771c8;">River</span></a></strong>, our latest Prossimo initiative focused on building a high performance and memory safe reverse proxy.<br>
									<br>
									Reverse proxy software plays a crucial role in Internet deployments, yet most existing software still relies on memory unsafe languages. River aims to dramatically reduce the risk of memory safety vulnerabilities, while also delivering top-tier performance.<br>
									<br>
									<strong>Cloudflare, Shopify, </strong>and<strong> Chainguard</strong> are collaborating with us on the project; if your organization would like to contribute to the effort, please<a href="mailto:donate@letsencrypt.org" style="text-decoration:none;color:#3771c8;"><strong>&nbsp;reply to this email!</strong></a></p>

									<h3 style="line-height: 1.38; margin-bottom: 8px;"><span style="color:#003a70;">Reducing Dependencies in sudo-rs</span></h3>

									<p>The <a href="www.memorysafety.org/initiative/sudo-su/" style="text-decoration:none;color:#3771c8;"><strong>sudo and su</strong></a> utilities mediate a critical privilege boundary on just about every open source operating system that powers the Internet. Unfortunately, these utilities have a long history of memory safety issues. That’s why Prossimo has been partnering with <strong>Tweede golf</strong> and <strong>Ferrous Systems</strong> to reimplement these ubiquitous utilities in the memory safe language Rust.<br>
									<br>
									<strong><a href="www.memorysafety.org/blog/reducing-dependencies-in-sudo/" style="text-decoration: none; color: rgb(55, 113, 200);">Check out the latest updates</a></strong> in our blog post by Ruben Nijveld of Tweede golf, celebrating a significant milestone in our sudo-rs work: <strong>reducing its dependencies from 135 to just three!</strong></p>

									<h3><span style="color:#003a70;">New Intermediate Certificates</span></h3>
									Last Wednesday, Let’s Encrypt generated 10 new Intermediate CA Key Pairs, and issued 15 new Intermediate CA Certificates containing the new public keys. Our new batch of intermediates differ from the ones we issued in 2020 in two key🔑 ways:<img align="right" alt="" border="0" height="70" src="/images/newsletters/2024/2024-03-21-isrg-brighter-bytes-march-2024-newsletter/image-05.png" style="width: 70px; height: 70px; float: right; margin: 5px; border-width: 0px; border-style: solid;" width="70">
									<ul>
										<li>We shortened the lifetime from five years to three</li>
										<li>We issued five of each key type (RSA and ECDSA) instead of two</li>
									</ul>
									The new intermediate certificates provide smaller, more efficient certificate chains to Let’s Encrypt Subscribers, enhancing the overall online experience in terms of speed, security, and accessibility. <strong><a href="letsencrypt.org/2024/03/19/new-intermediate-certificates" style="text-decoration:none;color:#3771c8;">Read all about these exciting updates</a></strong> in our latest blog post.<br>

<h3 style="line-height: 1.38; margin-bottom: 8px;"><span style="color:#003a70;">Rustls TLS library using AWS Libcrypto, now has FIPS support&nbsp;</span></h3>

<p>We know that widespread adoption of memory safe TLS libraries is a critical step towards a more secure and privacy-respecting Internet. That’s why the recent shift to using AWS Libcrypto for Rust (aws-lc-rs) as the default cryptography provider for the Rustls TLS library is so exciting! This change, along with the option to enable FIPS support, marks a significant step forward in making TLS implementations safer and more adaptable for a wide range of organizational needs.&nbsp;<br>
<br>
For more information on these advancements, including the introduction of comprehensive benchmarking for Rustls and plans for an OpenSSL compatibility layer, <strong><a href="www.memorysafety.org/blog/rustls-with-aws-crypto-back-end-and-fips/" style="color:#3771c8;text-decoration:none;">read the full blog post.</a></strong></p>

<h3 style="line-height: 1.38; margin-bottom: 8px;"><span style="color:#003a70;">2024: The Year of Memory Safety</span></h3>

<p>Last month, the White House’s Office of the National Cyber Director <a href="www.whitehouse.gov/oncd/briefing-room/2024/02/26/memory-safety-fact-sheet/" style="color:#3771c8;text-decoration:none;"><strong>issued a report</strong></a> calling for large-scale adoption of memory safe programming languages. The report clearly echoes Prossimo’s outlook, and even references our strategic approach in the full report.<br>
<br>
<strong>Here are the key takeaways:</strong></p>

<ul>
	<li>Making memory safe choices now addresses a solvable security issue</li>
	<li>Evidence shows that memory safe languages enhance digital security and protect individual privacy</li>
	<li>We should prioritize rewriting security-sensitive functions first (rather than everything, everywhere, all at once)</li>
</ul>

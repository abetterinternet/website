<h3><span style="color:#003a70;">ACME Renewal Information (ARI) Published as RFC 9773</span></h3>
We are proud to share that the ACME Renewal Information (ARI) extension is now published as RFC 9773. ARI allows a CA to advise a software client to perform an early renewal of a certificate that the client would have otherwise anticipated did not need to be renewed yet, broadly because the CA knows that an early renewal is helpful, or necessary, in particular circumstances. Make sure your ACME client is up to date and check whether it supports ARI.&nbsp;<br>
<br>
<a href="https://outreach.abetterinternet.org/e/1011011/2025-09-16-ari-rfc/3sk9d/%%email_id%%/h/%%internal.tracker_url_hash%%">Continue reading the full blog post for more.</a><br>
&nbsp;
<h3><span style="color:#003a70;">Native ACME Support Comes to NGINX</span></h3>
NGINX now includes built-in ACME support, powered by the new ngx_http_acme module written in Rust. This integration enables seamless certificate issuance and renewal directly within NGINX. NGINX is the most widely used web server or proxy, so this change will improve certificate issuance for millions of users. We’re grateful to F5 NGINX for sponsoring Let’s Encrypt’s work serving nearly 700 million websites.<br>
<br>
<a href="https://outreach.abetterinternet.org/e/1011011/25-09-11-native-acme-for-nginx/3sk9h/%%email_id%%/h/%%internal.tracker_url_hash%%">Continue reading the full blog post for more.</a><br>
&nbsp;
<h3><span style="color:#003a70;">OCSP Service Reaches End of Life</span></h3>
On August 6, 2025, we officially retired our OCSP service. Revocation information is now provided exclusively through Certificate Revocation Lists (CRLs).<br>
<br>
<a href="https://outreach.abetterinternet.org/e/1011011/ervice-has-reached-end-of-life/3sk9l/%%email_id%%/h/%%internal.tracker_url_hash%%">Continue reading the full blog post for more.</a><br>
&nbsp;
<h3><span style="color:#003a70;">RFC 6962 Certificate Transparency Logs EOL</span></h3>
Wet announced our plan to deprecate RFC 6962 CT logs. They’ll become read-only on November 30, 2025, and be fully retired by February 28, 2026. Future logging will transition to the new Static CT API.<br>
<br>
<a href="https://outreach.abetterinternet.org/e/1011011/2025-08-14-rfc-6962-logs-eol/3sk9p/%%email_id%%/h/%%internal.tracker_url_hash%%">Continue reading the full blog post for more.</a><br>

<h3><span style="color:#003a70;">Improving Error Handling in Rustls for Better Usability</span></h3>
The latest updates to Rustls introduce more robust and user-friendly error handling, making it easier for developers to identify and address issues. These improvements strengthen the usability of Rustls and reduce frustration when things go wrong.<br>
<br>
<a href="https://outreach.abetterinternet.org/e/1011011/blog-rustls-error-handling-/3sk9s/%%email_id%%/h/%%internal.tracker_url_hash%%">Continue reading the full blog post for more</a>.

<h3><br>
<span style="color:#003a70;">Rustls Joins the Rust Foundation’s Rust Innovation Lab</span></h3>
​​​​​​The Rustls TLS library has a new home! It is the inaugural project hosted by the Rust Foundation's Rust Innovation Labs. We're proud that Prossimo raised over $1M to support the development of Rustls over the last few years and today it's an excellent choice for TLS - memory safe, high performance, and a stable of features. Congratulations to the creator of Rustls, Joe Birr-Pixton, and the other current maintainers for making this big step!<br>
<br>
<a href="https://outreach.abetterinternet.org/e/1011011/oundation-rust-innovation-lab-/3sk9w/%%email_id%%/h/%%internal.tracker_url_hash%%">Continue reading the full blog post for more</a>.<br>
